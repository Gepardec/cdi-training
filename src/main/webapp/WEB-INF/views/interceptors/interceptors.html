An interceptor is either registered via the a beans.xml or via the @Priority annotation.

Here you find the spec for Interceptor Bindings

1. Interceptor enabled via @Priority

Interceptors enabled with @Priority are enabled globally in all CDI modules.
If all interceptors define the same priority then the order they are executed is not deterministic.

Your Task:

Ensure that the interceptors declared on the rest resource at.gepardec.cditraining.interceptors.InterceptorsController, are invoked in the following order:
FirstInterceptor
SecondInterceptor

Important!!
Ensure that no interceptor is enabled in the beans.xml

2. Interceptor enabled via beans.xml

Interceptors enabled with a beans.xml are enabled only in the declaring CDI module. With this approach you have not problems with ordering.

Your Task:

Ensure that the interceptors declared on the rest resource at.gepardec.cditraining.interceptors.InterceptorsController, are invoked in the following order:
SecondInterceptor
FirstInterceptor

Important!!
Ensure that no interceptor is enabled via @Priority annotations forehand

4. Interceptor with bindings

Interceptors can hold literal values as any other annotation.
These values can be used to select an interceptor implementation instead of introducing a new interceptor annotation.

Your Task:

Implement two interceptors, which log simple message to the console, for the interceptor binding at.gepardec.cditraining.interceptors.BindingInterceptor and enhance the interceptor binding so that you can select one of your implementations on the annotated method at.gepardec.cditraining.interceptors.InterceptorsController.binding().
