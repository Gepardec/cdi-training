Normally we define injections of beans during compile time which get resolved during runtime

But what if the information what bean to use is dynamic and can change during runtime?

Imagine an interface which has multiple implementations, although the current implementation to use is decided during runtime and therefore cannot be defined during compile time

So, wee need to implement a programmatic lookup to resolve the beans during runtime when we know what implementation we need to use

Important!!
Be ware that if the beans are @Dependent then we nee to cleanup them, otherwise we would cause a memory leak
The solution is to implement a factory class which encapsulates the programmatic lookup and bean instance management
