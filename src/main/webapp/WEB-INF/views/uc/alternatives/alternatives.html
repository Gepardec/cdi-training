See the package at.gepardec.cditraining.uc.alternatives.* for the relevant classes. Especially take a look at the comments.

Alternatives beans can be defined in two ways:

  1. Via the @Alternative annotation on the alternative implementation class along with either an beans.xml entry or a @priority annotation.
  2. Via an custom stereotype annotation which contains the annotations @Stereotype @Alternative and an entry in the beans.xml. If activated all beans annotated with stereotype annotation are activated.

Compared to @Specializes annotation, the @Alternative annotation doesn't replace the original bean, which is still accessible by it's type (clazz) or qualifier.

Via the @Priority annotation we can provide alternatives for beans of other CDI bean archives, even from third party archives.

The alternative implementation is annotated with @Alternative and is enabled via the the beans.xml or the @Priority annotation.
