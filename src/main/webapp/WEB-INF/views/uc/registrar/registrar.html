See the package at.gepardec.cditraining.uc.registrar.* for the relevant classes. Especially take a look at the comments.

Just firing an asynchronous event or an asynchronous REST/SOAP call and forgetting it mostly not possible. What if we need to report the success/failed events/requests periodically to a metric backend?

In this example we use an registrar which holds the success/failed events for further usage. This bean abstracts the producer of the data (event observer) from the consumer or the data (maybe a metrics reporter)

The registrar collects the data and makes it available to other CDI beans.
