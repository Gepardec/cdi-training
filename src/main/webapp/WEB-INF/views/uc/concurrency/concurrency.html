Concurrency in CDI is a little bit difficult, because only @ApplicationScoped and @Dependent beans are available on other threads

We have an web application, so our concurrent workflows are started from an active request, which will also provide state for the concurrent execution.

So we have to do a little trick to provide all request related beans as dependent scoped beans to the concurrent executed bean.

We do that by managing the state indirectly via a request scoped producer bean
