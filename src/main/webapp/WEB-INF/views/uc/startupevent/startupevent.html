The specification for CDI Events

CDI Events are used couple parts of an application in decoupled manner, where all application parts are only coupled via the event type and nothing more. Events are used to notify observers about that something has happened.

A use-case for using CDI Events would be to notify modules of an application about the startup of the application, that the application is ready, or that the application is being shutdown, so that the application modules can act on those events.

Search the startup logs for the following logs:

2021-02-08 07:58:32 INFO  StartupWebListener - Notifying Observers about startup
2021-02-08 07:58:32 INFO  StartupObserver - Startup observed by StartupObserver
2021-02-08 07:58:32 INFO  StartupWebListener - Notified Observers: StartupObserver

See the package at.gepardec.cditraining.uc.startupevent.* for the relevant classes. Especially take a look at the comments

The ServletContextListener implementation is used to fire an custom event type when the servlet container is initialized, which means the container is up and running.
