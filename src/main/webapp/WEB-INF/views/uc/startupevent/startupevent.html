See the package at.gepardec.cditraining.uc.startupevent.* for the relevant classes. Especially take a look at the comments

CDI has no native support for a StartupEvent which gets fired when the container is up and running. But this is not hard to implement at all

In this example we have created our own StartupEvent type (just an ordinary java class) and a ServletContextListener implementation to register for a servlet container initialized event. When the servlet container is up and running then the CDI container is as well

The ServletContextListener implementation is used to fire our event when the servlet container is up and running to notify the registered observers
