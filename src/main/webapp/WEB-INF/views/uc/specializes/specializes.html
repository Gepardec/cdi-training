See the package at.gepardec.cditraining.uc.specializes.* for the relevant classes. Especially take a look at the comments.

Specialized beans can be defined in two ways:

  1. Via the @Specialized annotation on the specialized implementation class
  2. Via the @Alternative annotation on the specialized implementation class along with an beans.xml entry or an @Priority annotation.

Compared to alternative beans, specialized beans replace the original bean, no matter with what approach a specialized bean has been defined.

When using a stereotype annotation, then we could just enable the stereotype annotation as an alternative (same for specialized) and all annotated beans would be activated.

Via the @Priority annotation we can provide specialized beans of other CDI bean archives, even from third party archives as you can see in the class at.gepardec.cditraining.TemplateEngineProducer.

The alternative implementation is annotated with @Alternative and is enabled via the the beans.xml or the @Priority annotation.
