If a constructor is used for injection, then you need to mark it.

Depending on the CDI implementation you use, it could be necessary to provide an empty constructor, otherwise your beans would cause exceptions.

In your case it was a JAX-RS implementation related problem, actually caused by Resteasy, which requires an empty constructor on a rest resource implementation. See this RedHat Issue for further information
