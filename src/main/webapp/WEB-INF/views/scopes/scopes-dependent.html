The specification for CDI Scopes and Contexts Scopes and Contexts

The @Dependent scope is a so called pseudo-scope which is always active, which means there is no lazy instantiation. A dependent CDI Bean is always owned by the CDI Bean which injected it, therefore, the dependent CDI Bean lives as long as the owning CDI Bean. But this is a little tricky when dependent CDI Beans are created programmatically via javax.enterprise.inject.Instance<T>, which we will see when we come to the use-cases.

When your scopes of the Basic Scopes and Mixed Scopes examples are set properly, then you should see the values changing as described.

Important!!
Ensure that you open an private/incognito Tab or a different browser to ensure different http sessions
request.dependent.value never changes
session.dependent.value starts new in a window
application.dependent.value always changes
